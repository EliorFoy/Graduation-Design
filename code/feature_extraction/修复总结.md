## ✅ 已修复的关键错误

### 1. `csp_feature.py` - CSP 特征提取

#### ❌ 错误 1: 多分类 CSP 使用差矩阵而非广义特征值
**问题**: 
```python
P_diff = P_i - P_others
eigenvals, eigenvecs = eigh(P_diff)  # 错误！
```

**修复**:
```python
# 使用广义特征值分解求解 CSP
# 目标：max w^T P_i w / w^T P_others w
# 等价于求解广义特征值问题：P_i v = λ P_others v
try:
    eigenvals, eigenvecs = eigh(P_i, P_others)
except np.linalg.LinAlgError:
    # 如果 P_others 奇异，添加小的正则化项
    P_others_reg = P_others + 1e-6 * np.eye(n_channels)
    eigenvals, eigenvecs = eigh(P_i, P_others_reg)
```

**影响**: 这是最严重的错误，会导致多分类 CSP 滤波器不是最优解，严重影响分类性能。

---

#### ❌ 错误 2: 正则化添加位置不当
**问题**:
```python
cov = cov / np.trace(cov)      # 先迹归一化
if reg_param > 0:
    cov += reg_param * np.eye(n_channels)  # 再加单位阵（错误！）
```

**修复**:
```python
# 先添加正则化，再进行迹归一化
if reg_param > 0:
    cov += reg_param * np.eye(n_channels)
cov = cov / np.trace(cov)
```

**影响**: 迹归一化后协方差矩阵的迹固定为1，再加入单位阵会改变矩阵结构和数值稳定性。

---

#### ❌ 错误 3: 只取最大化方差的滤波器
**问题**:
```python
# 每个类只取前 n_comp_per_class 个最大特征值对应的滤波器
W_csp_list.append(V[:n_comp_per_class, :])
```

**修复**:
```python
# 同时保留最大和最小的滤波器（增强判别力）
n_comp_half = max(1, n_comp_per_class // 2)
top_count = n_comp_half + (1 if n_comp_per_class % 2 != 0 else 0)
bottom_count = n_comp_half

top_idx = list(range(top_count))
bottom_idx = list(range(V.shape[0] - bottom_count, V.shape[0]))
selected_idx = top_idx + bottom_idx

W_csp_list.append(V[selected_idx, :])
```

**影响**: CSP 的核心思想是同时提取对某类敏感（大方差）和对其他类不敏感（小方差）的滤波器，丢失最小滤波器会降低判别力。

---

#### ⚠️ 优化 1: 二分类奇数成分提示
添加了当 `n_components` 为奇数时的警告信息，明确说明如何分配最大/最小滤波器数量。

#### ⚠️ 优化 2: 白化数值稳定性
```python
# 处理极小特征值，避免除零
eigenvalues[eigenvalues < 1e-10] = 0
D_inv_sqrt = np.diag(1.0 / np.sqrt(eigenvalues + 1e-10))
```

### 2. `wavelet_feature.py` - 小波特征提取

#### ❌ 错误 1: CWT 频率标度计算错误
**问题**:
```python
scales = pywt.frequency2scale(wavelet, freq / sfreq)  # 错误！
```

**修复**:
```python
# pywt.frequency2scale 的第二个参数应为归一化频率 f/f_nyquist
nyquist = sfreq / 2.0
scale = pywt.frequency2scale(wavelet, freq / nyquist)
```

**影响**: 这会导致连续小波变换得到完全错误的尺度，频段功率计算全部错误。

---

#### ❌ 错误 2: `pywt.upcoef` 已移除
**问题**:
```python
cA_rec = pywt.upcoef('a', coeffs[0], wavelet, level=level, take=len(signal))
# AttributeError: module 'pywt' has no attribute 'upcoef'
```

**修复**:
```python
# 使用 waverec 替代已移除的 upcoef
coeffs_A = [coeffs[0]] + [np.zeros_like(cD) for cD in coeffs[1:]]
cA_rec = pywt.waverec(coeffs_A, wavelet)[:len(signal)]

# 重建单级细节系数
coeffs_D = [np.zeros_like(coeffs[0])]
for j in range(1, len(coeffs)):
    if j == i:
        coeffs_D.append(coeffs[j])
    else:
        coeffs_D.append(np.zeros_like(coeffs[j]))
cD_rec = pywt.waverec(coeffs_D, wavelet)[:len(signal)]
```

**影响**: 新版 PyWavelets (v1.0+) 已移除 `upcoef`，原代码会直接崩溃。

---

#### ⚠️ 优化 1: 低频函数标记为低效
在 `extract_band_power_features()` 的文档字符串中添加警告：

```python
⚠️  注意：此函数计算效率极低，仅用于演示。推荐使用 extract_wavelet_energy_features()。
```

该函数对每个试次、每个通道、每个频段在 10 个频率点分别调用 CWT，计算量巨大。

#### ⚠️ 优化 2: 频段命名自适应
**问题**: 原代码硬编码特定采样率（250 Hz）的频段映射：
```python
if i == 1: name = 'gamma'

# ...
```

**修复**: 基于实际频率动态命名：
```python
if f_low >= 30: name = 'gamma'

elif f_low >= 8: name = 'mu+alpha'
elif f_low >= 4: name = 'theta'
else: name = 'delta'
```

## 📝 注意事项

1. **CSP 广义特征值分解可能失败**
   - 如果 `P_others` 矩阵奇异，会自动添加小的正则化项 (1e-6)
   - 这种情况通常出现在数据质量极差时

2. **小波分解可视化**
   - `plot_wavelet_decomposition()` 现在使用 `waverec` 重建系数
   - 重建信号长度可能与原始信号略有差异（边界效应），已用 `[:len(signal)]` 截断

3. **特征维度平衡**
   - CSP 特征通常只有 4 维，而小波特征有 110 维（22 通道 × 5 层）
   - 融合特征中小波的贡献占主导，这是正常的
   - 如需平衡，可在 SVM Pipeline 中使用 StandardScaler
