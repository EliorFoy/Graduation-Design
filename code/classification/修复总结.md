## ✅ 已修复的关键问题

### 0. **cross_val_score 与 Epochs 对象兼容性问题（运行时崩溃）**

#### ❌ 问题描述
`train_eeg_svm_pipeline()` 直接传入 MNE `Epochs` 对象给 `cross_val_score`，导致 sklearn 内部索引时维度丢失：

```python
X = epochs_or_data  # Epochs 对象
cv_scores = cross_val_score(pipeline, X, y, cv=cv, scoring='accuracy')
# ❌ StratifiedKFold 索引后可能返回二维数组，CSP 收到 (n_trials, n_features) 而非 (n_trials, n_channels, n_times)
# ValueError: too many values to unpack (expected 3)
```

**根本原因**：sklearn 的 `safe_indexing` 对 `Epochs` 对象的切片行为不符合预期，导致 Pipeline 中的 CSP 收到错误维度的数据。

#### ✅ 修复方案
在调用 `cross_val_score` 前显式将 `Epochs` 转为三维 numpy 数组：

```python
# 【关键修复】显式将 Epochs 转为三维数组，避免 cross_val_score 索引时维度丢失
if hasattr(epochs_or_data, 'get_data'):
    X = epochs_or_data.get_data()  # 三维数组 (n_trials, n_channels, n_times)
    print(f"   - 输入数据类型：Epochs → numpy.ndarray")
    print(f"   - 数据形状：{X.shape}")
else:
    X = np.asarray(epochs_or_data)
    print(f"   - 输入数据类型：numpy.ndarray")
    print(f"   - 数据形状：{X.shape}")

# 确认形状为 3D
if X.ndim != 3:
    raise ValueError(f"CSP 需要 3D 输入 (n_trials, n_channels, n_times)，但收到 {X.ndim}D 数据，形状：{X.shape}")

pipeline.fit(X, y)  # Pipeline 内部的 EpochsToArray 会直接返回该数组
```

**影响**: 这是**阻断性错误**，不修复则程序完全无法运行。修复后确保数据维度正确传递。

### 1. **缺失特征标准化（最严重）****

#### ❌ 问题描述
`train_svm_classifier()` 和 `optimize_svm_hyperparameters()` 直接对原始特征训练 SVM，未包含标准化步骤。SVM（特别是 RBF 核）对特征尺度极其敏感，未标准化的特征会导致：
- CSP 特征（4 维）和小波特征（110 维）尺度差异巨大
- 模型性能严重下降
- 用户难以排查问题根源

#### ✅ 修复方案
为两个函数添加 `auto_scale=True` 参数（默认启用）：

```python
def train_svm_classifier(X, y, cv_folds=10, kernel='rbf', random_state=42, auto_scale=True):
    if auto_scale:
        from sklearn.pipeline import Pipeline as SklearnPipeline
        from sklearn.preprocessing import StandardScaler
        
        clf = SklearnPipeline([
            ('scaler', StandardScaler()),
            ('svm', SVC(kernel=kernel, random_state=random_state))
        ])
        print("   ✅ 已启用自动标准化（StandardScaler）")
    else:
        clf = SVC(kernel=kernel, random_state=random_state)
        print("   ⚠️  警告：未启用自动标准化，请确保输入特征已标准化")
```

**影响**: 这是最重要的修复，确保用户不会因忘记标准化而得到错误结果。

### 2. **Pipeline 重复转换**

#### ❌ 问题描述
```python
X = epochs_or_data.get_data() if hasattr(epochs_or_data, 'get_data') else epochs_or_data
pipeline.fit(X, y)  # Pipeline 第一步又是 EpochsToArray
```
在传入 Pipeline 前就提取了数组，但 Pipeline 的第一步 `EpochsToArray` 会再次处理，造成不必要的冗余。

#### ✅ 修复方案
直接传入原始数据，让 Pipeline 内部处理转换：

```python
# 【优化】直接传入原始数据，让 Pipeline 内部的 EpochsToArray 处理转换
X = epochs_or_data
y = np.asarray(y)
print(f"   - 输入数据类型：{type(X).__name__}")
if hasattr(X, 'get_data'):
    print(f"   - Epochs 形状：{X.get_data().shape}")
else:
    print(f"   - 数据形状：{np.asarray(X).shape}")

pipeline.fit(X, y)  # Pipeline 会自动调用 EpochsToArray
```

**影响**: 代码更简洁，避免重复操作，逻辑更清晰。

### 3. **网格搜索参数可视化不完善**

#### ❌ 问题描述
`plot_cv_results()` 假设参数网格只包含 RBF 核和 gamma 参数，当遇到以下情况时会失败：
- 使用 `kernel='linear'`（无 gamma 参数）
- gamma 值混合字符串（'scale'）和浮点数
- 参数维度超过 2 维

#### ✅ 修复方案
```python
# 【优化】仅当参数网格包含 RBF 核且有 gamma 参数时才绘制热力图
has_rbf = any('gamma' in str(p) or p.get('kernel') in [None, 'rbf'] for p in params)

# 提取 C 的值
C_values = sorted(list(set([p['C'] for p in params if 'C' in p])))

gamma_values = []

if has_rbf:
    # 【修复】更健壮的 gamma 值提取和排序
    gamma_raw = [p.get('gamma') for p in params if 'gamma' in p and p.get('gamma') is not None]
    if gamma_raw:
        # 分离数值型和字符串型 gamma
        numeric_gammas = sorted([g for g in gamma_raw if isinstance(g, (int, float))])
        string_gammas = sorted([str(g) for g in gamma_raw if not isinstance(g, (int, float))])
        gamma_values = numeric_gammas + string_gammas

if len(C_values) > 1 and len(gamma_values) > 1:
    # 绘制 C-gamma 热力图
    ...
```

**改进**:
- 检查是否包含 RBF 核
- 安全提取 C 和 gamma 值
- 分离数值型和字符串型 gamma，避免排序错误
- 仅在条件满足时绘制热力图

### 4. **混淆矩阵归一化除零风险**

#### ❌ 问题描述
```python
cm_normalized = cm.astype('float') / cm.sum(axis=1)[:, np.newaxis] * 100
```
如果某类别在测试集中未出现（分母为 0），会产生 NaN 或 Inf，导致可视化异常。

#### ✅ 修复方案
```python
# 【优化】安全归一化，避免除零错误
row_sums = cm.sum(axis=1)
cm_normalized = np.zeros_like(cm, dtype=float)
for i in range(n_classes):
    if row_sums[i] > 0:
        cm_normalized[i] = cm[i].astype('float') / row_sums[i] * 100
    else:
        print(f"   ⚠️  警告：类别 {i} 在真实标签中未出现，归一化结果为 NaN")
```

**影响**: 增强鲁棒性，避免因边界情况导致的崩溃。

## 📝 注意事项

### 1. **向后兼容性**
- `auto_scale=True` 是默认值，旧代码若已手动标准化，建议设置 `auto_scale=False` 避免双重标准化
- 返回值顺序保持不变，不影响现有调用

### 2. **Pipeline vs 直接分类器**
- `train_svm_classifier` 在 `auto_scale=True` 时返回 Pipeline，否则返回 SVC
- 使用时需注意类型差异：
  ```python
  clf, _, _ = train_svm_classifier(X, y, auto_scale=True)
  # clf 是 Pipeline，可通过 clf.named_steps['svm'] 访问 SVC
  ```

### 3. **网格搜索参数前缀**
- 启用 `auto_scale` 时，参数网格会自动添加 `'svm__'` 前缀
- 自定义参数网格时无需手动添加前缀

### 4. **可视化限制**
- `plot_cv_results` 仅支持二维参数（C 和 gamma）的热力图
- 若需更高维可视化，建议使用其他工具（如 Plotly 平行坐标图）
