## 1. 数据集结构

### BCIC IV-2a 数据集基本信息

| 属性 | 值 |
|------|-----|
| **实验类型** | 运动想象（Motor Imagery, MI） |
| **被试数量** | 9 人（A01-A09） |
| **通道数量** | 22 个 EEG + 3 个 EOG |
| **采样率** | 250 Hz |
| **任务类别** | 4 类运动想象 |

### 4 类运动想象任务

| 事件代码 | 想象动作 | 激活大脑区域 |
|---------|---------|-------------|
| **769** | 左手运动想象 | 右侧运动皮层 (C4) |
| **770** | 右手运动想象 | 左侧运动皮层 (C3) |
| **771** | 双脚运动想象 | 顶部运动皮层 (Cz) |
| **772** | 舌头运动想象 | 底部运动皮层 (FCz) |

### 文件命名规则

```
A##X.gdf

其中:
- A: 被试标识
- ##: 被试编号 (01-09)
- X: 数据类型
  - T: Training data（训练数据，带标签）
  - E: Evaluation data（评估数据，用于测试）

示例:
- A01T.gdf → 被试 1 的训练数据
- A01E.gdf → 被试 1 的测试数据
```

## 2. 试次（Trial）概念

### 核心区别：动作类型 vs 试次数

```
❌ 错误理解:
  4 个动作 = 4 个试次
  
✅ 正确理解:
  4 个动作类型 × 每个动作重复 72 次 = 288 个试次
```

### 为什么要重复 72 次？

| 重复次数 | 总试次 | 统计效力 | 推荐度 |
|---------|--------|---------|--------|
| 每类 1 次 | 4 次 | ⭐⭐ 太低 | ❌ 无法训练 |
| 每类 10 次 | 40 次 | ⭐⭐ 不足 | ❌ |
| **每类 72 次** | **288 次** | ⭐⭐⭐⭐⭐ **很高** | **✅✅✅** |
| 每类 100 次 | 400 次 | ⭐⭐⭐⭐⭐ 极高 | ⭐⭐ 被试太累 |

**72 次的优势：**
1. ✅ 满足统计学要求（中心极限定理：n>30）
2. ✅ 支持 10 折交叉验证
3. ✅ 避免过拟合（样本数 > 特征维度）
4. ✅ 实验时长适中（约 34 分钟）

### 单个试次的时间结构

```
时间线:
0s     2s          4s         6s    7s
│------│-----------│----------│------│
│      │           │          │      │
固定   蜂鸣声      箭头       执行   休息
十字            (→出现)    (右手)  间隔

├─ 0.0s - 2.0s:  静息状态（注视"+"）
├─ 2.0s:        提示音（蜂鸣声）
├─ 2.0s - 3.25s: 屏幕显示箭头提示
├─ 3.25s - 6.0s: 被试执行运动想象 ⭐ ← 在这期间写入事件标记
└─ 6.0s - 7.0s:  休息间隔

单次试次时长：约 7 秒
有效任务段：3.25s - 6.0s（通常截取 0-4 秒作为 Epoch）
```

### 数据结构变化

```python
# 原始连续数据 (Raw)
raw.shape = (25 通道，约 600,000 时间点)
# 24-34 分钟的连续记录

# 分段后数据 (Epochs)
epochs.shape = (288, 22, 1000)
               ↑    ↑   ↑
             试次数 通道 4 秒×250Hz

# 物理意义:
# - 288 个独立的试次（样本）
# - 每个试次包含 22 个通道的脑电信号
# - 每个试次时长 4 秒（1000 个采样点）
```

## 3. 事件标记系统

### GDF 文件的内部结构

GDF 文件包含两部分：

```
┌─────────────────────────────────────────────┐
│  1. 连续的 EEG/EOG 信号数据                  │
│  shape: (25, n_samples)                     │
│  - 22 个 EEG 通道                            │
│  - 3 个 EOG 通道                             │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│  2. 事件标记列表 (Annotations)              │
│  shape: (n_events, 3)                       │
│  每行格式：[时间点数，0, 事件代码]           │
└─────────────────────────────────────────────┘
```

### 事件标记示例

```python
events 数组示例:
[
    [500,    0,  768],  ← 第 500 点，开始提示
    [1000,   0,  769],  ← 第 1000 点，左手任务
    [3500,   0,  770],  ← 第 3500 点，右手任务
    [6000,   0,  771],  ← 第 6000 点，双脚任务
    [8500,   0,  772],  ← 第 8500 点，舌头任务
    [11000,  0, 1023],  ← 第 11000 点，伪迹标记
    ...
]
```

### 事件代码含义

| 代码 | 含义 | 类型 | 数量 |
|------|------|------|------|
| 768 | 开始提示（十字出现） | 提示代码 | 288 |
| 769 | 左手想象 | **任务代码** | 72 |
| 770 | 右手想象 | **任务代码** | 72 |
| 771 | 双脚想象 | **任务代码** | 72 |
| 772 | 舌头想象 | **任务代码** | 72 |
| 783 | 试次结束 | 提示代码 | 288 |
| 1023 | 严重伪迹（专家手动标记） | **伪迹标记** | 可变 |

### 🔑 重要说明：1023 伪迹标记

**1023 的性质：**
- ❌ **1023 不是运动想象任务**，而是实验专家在数据采集时手动标记的"警告标签"
- ❌ **1023 不包含在 288 个试次中**，它是额外的元数据（关于数据的数据）
- ✅ **1023 是贴在部分试次上的"质量警告"**，表示这个时间段的 EEG 信号被严重污染

**为什么需要 1023 标记？**
```
实验现场:
时间轴:
0s     2s          4s         6s    7s
│------│-----------│----------│------│
│      │           │          │      │
固定   蜂鸣声      箭头       执行   休息
十字            (→出现)    (右手)  间隔

                    ↓
              被试突然咳嗽了一下！
              EEG 信号出现大幅值尖峰 (>500μV)
      
                    ↓
              实验专家立即按下标记按钮
              在 GDF 文件中写入事件代码 1023
              
                    ↓
              这个试次被标记为"严重伪迹"
              后续处理时应该剔除
```

**1023 与 288 个试次的关系：**
```
类比理解:

想象你在批改 288 份试卷：

✅ 正式试卷（288 份）:
   - 语文试卷 × 72 份  (代码 769)
   - 数学试卷 × 72 份  (代码 770)
   - 英语试卷 × 72 份  (代码 771)
   - 物理试卷 × 72 份  (代码 772)
   
⚠️ 污损标记（1023）:
   "这 8 份试卷被墨水弄脏了，不要评分！"
   
   ↓
   
   这 8 份污损的试卷 **已经包含在** 288 份试卷中了
   而不是额外的 296 份（288 + 8）

同样地:
- 288 个正式试次 = 288 份"试卷"
- 1023 标记 = 贴在部分试卷上的"污损警告"
- 被标记的试次仍然是 288 个之一，只是质量差被剔除了
```

**实际数据结构示例：**

## 4. 分段处理的重要性

### 什么是分段（Epoching）？

```
连续 EEG 数据（Continuous Raw Data）:
┌─────────────────────────────────────────┐
│ ──────┬───┬─────┬───┬─────┬───┬────── │
│       ↑   ↑     ↑   ↑     ↑   ↑        │
│      T1  T2    T3  T4    T5  T6        │
└─────────────────────────────────────────┘
          ↓ 根据事件标记切割 ↓
分段后的 Epochs:
┌──────────┐  ┌──────────┐  ┌──────────┐
│  Trial 1 │  │  Trial 2 │  │  Trial 3 │
│  0-4 秒  │  │  0-4 秒  │  │  0-4 秒  │
│  (左手)  │  │  (右手)  │  │  (双脚)  │
└──────────┘  └──────────┘  └──────────┘
```

### 为什么必须分段？

#### ❌ 不分段的问题

```python
# 连续数据形状
raw.shape = (22, 360000)

# 问题 1: 如何打标签？
y = ???  
# 360000 个时间点，每个点的标签是什么？
# [?, ?, ?, ?, ?, ...] ← 无法确定！

# 问题 2: 如何定义"一个样本"？
# 机器学习的标准格式：X ∈ ℝ^(N × F)
#                      ↑   ↑ ↑
#                    样本数 特征数
# 
# 但 continuous data 是：(22, 360000)
#                       ↑   ↑
#                     通道 时间
# → 完全不符合机器学习输入格式！
```

#### ✅ 分段后的优势

```python
# 分段后形状
epochs.shape = (288, 22, 1000)
               ↑    ↑   ↑
             样本数 通道 时间点

# 标签自动获取
y = epochs.events[:, 2]  # (288,)
# [769, 770, 769, 771, ...]
#  左手，右手，左手，双脚...
#  ← 每个试次一个标签，完美对应！

# 符合机器学习格式
X = epochs.get_data()  # (288, 22, 1000)
X_flat = X.reshape(288, -1)  # (288, 22000)
#                              ↑    ↑
#                            样本数 特征数
```

### 分段的物理意义

```
类比：图像分类任务

❌ 不分段:
   给模型看一整部电影 → "这是喜剧还是悲剧？"
   （信息混杂，无法判断）

✅ 分段:
   给模型看 288 张截图 → "这张图是笑还是哭？"
   （每张图一个标签，清晰明确）


EEG 分段也是同样的道理:
- 从 24 分钟电影中提取 288 个"精彩片段"
- 每个片段 4 秒，对应一次完整的运动想象
- 每个片段有明确的标签（左手/右手/双脚/舌头）
```

## 5. 特征提取：分段 vs 连续

### CSP 特征提取对比

#### ✅ 方案 A：分段后的 CSP 特征提取（标准做法）

```python
# 输入：epochs (288, 22, 1000)

# Step 1: 按类别分组
epochs_left = epochs['769']   # (72, 22, 1000)
epochs_right = epochs['770']  # (72, 22, 1000)

# Step 2: 计算协方差矩阵
cov_left_list = []
for epoch in epochs_left:  # 遍历 72 个试次
    data = epoch.get_data()  # (22, 1000)
    cov = data @ data.T / np.trace(data @ data.T)
    cov_left_list.append(cov)

C_left = np.mean(cov_left_list, axis=0)  # (22, 22)

# Step 3: CSP 对角化，学习空间滤波器 W
W = csp_diagonalization(C_left, C_right)  # (22, 22)

# Step 4: 提取特征
X_csp = []
for epoch in epochs:  # 288 个试次
    data = epoch.get_data()  # (22, 1000)
    filtered = W.T @ data  # (22, 1000)
    feature = np.log(np.var(filtered[:2]) / np.var(filtered[-2:]))
    X_csp.append(feature)

X_csp = np.array(X_csp)  # (288, 4) ← 每个试次一个特征向量
```

**优势：**
- ✅ 标签准确（来自事件标记）
- ✅ 样本独立（每个试次是完整的）
- ✅ 信噪比高（基于完整试次计算协方差）
- ✅ 计算高效（288 次迭代）

#### ❌ 方案 B：连续数据的 CSP 特征提取（不推荐）

```python
# 输入：raw (22, 360000)

# Step 1: 滑动窗口切割
window_size = 1000  # 4 秒
step_size = 500     # 50% 重叠

windows = []
window_labels = []

for start in range(0, 360000 - window_size, step_size):
    end = start + window_size
    
    # ❌ 问题 1: 这个窗口的标签是什么？
    # 需要根据实验时间表推断（容易出错）
    center_time = (start + end) / 250
    
    if 10 <= center_time < 14:
        label = 'left'
    elif 20 <= center_time < 24:
        label = 'right'
    else:
        continue  # 休息时间，跳过
    
    window_data = raw[:, start:end]  # (22, 1000)
    windows.append(window_data)
    window_labels.append(label)

# ❌ 问题 2: 窗口之间有重叠，协方差估计有偏！
# ❌ 问题 3: 可能切分到任务边界，标签混乱
```

**问题总结：**
- ⚠️ 标签推断易错
- ⚠️ 样本不独立（窗口重叠）
- ⚠️ 可能包含任务转换期的噪声
- ⚠️ 计算冗余（大量重叠窗口）

### 小波特征提取对比

#### ✅ 分段后的小波特征提取

```python
# 输入：epochs (288, 22, 1000)

def extract_wavelet_features(epochs):
    all_features = []
    
    for epoch in epochs:  # 288 次循环
        epoch_data = epoch.get_data()  # (22, 1000)
        
        features_per_channel = []
        for channel_data in epoch_data:  # 22 次循环
            # 小波分解
            coeffs = wavedec(channel_data, 'db4', level=4)
            
            # 计算各层能量
            energies = [np.sum(c ** 2) for c in coeffs]
            normalized = [e / sum(energies) for e in energies]
            
            features_per_channel.extend(normalized)
        
        epoch_feature = np.array(features_per_channel)  # (110 维)
        all_features.append(epoch_feature)
    
    X_wavelet = np.array(all_features)  # (288, 110)
    return X_wavelet
```

**输出：**
```
epochs (288, 22, 1000)
  ↓ 逐个 epoch 处理
每个 epoch: 22 通道 × 5 层小波能量 = 110 维特征
  ↓
X_wavelet (288, 110)
每个试次一个特征向量 ← 符合机器学习格式！
```

### 特征提取对比总结表

| 维度 | **分段特征提取** ✅ | **连续特征提取** ❌ |
|------|------------------|-------------------|
| **输入数据** | `epochs (288, 22, 1000)` | `raw (22, 360000)` |
| **样本定义** | 一个试次 = 一个样本 | 一个窗口 = 一个样本 |
| **标签获取** | 直接从事件标记 | 需要推断（易错） |
| **特征形状** | `(288, n_features)` | `(n_windows, n_features)` |
| **CSP 协方差** | 基于完整试次（信噪比高） | 基于窗口（可能有偏） |
| **计算效率** | 288 次迭代 | 可能 700+ 次迭代 |
| **推荐使用** | ✅✅✅ 强烈推荐 | ❌ 不推荐 |

## 6. ICA 去噪与试次剔除的关系

### 两级伪迹去除策略

```
第一级：ICA 去噪（成分层面）
  → 去除稳定的、可分离的伪迹源
  → 眨眼、眼动、心跳
  
第二级：试次剔除（试次层面）
  → 去除瞬时的、残留的严重伪迹
  → 咳嗽、吞咽、电极接触不良
```

### 为什么 ICA 后还要剔除试次？

| 步骤 | 剔除对象 | 伪迹类型 | 操作层面 |
|------|---------|---------|---------|
| **ICA 去噪** | 独立成分 | **稳定的、周期性的**伪迹 | **成分层面** |
| **剔除试次** | 整个 Epoch | **瞬时的、偶发的**严重伪迹 | **试次层面** |

### ICA 能去除的伪迹

```
眨眼 (规律、稳定):
C3 ───╮ ╭───╮ ╭───╮ ╭────  ← ICA 可以识别并剔除

眼动 (与 EOG 相关):
C3 ───╱╲────╱╲────╱╲────  ← ICA 可以识别并剔除

心跳 (周期性):
C3 ──╱╲─╱╲─╱╲─╱╲─╱╲─  ← ICA 可以识别并剔除
```

### ICA 不能去除的伪迹

```
突然咳嗽 (瞬时、剧烈):
C3 ────────────╱╲────────  ← 峰值超过 500μV!
                  ↑
             ICA 难以建模

吞咽动作 (偶发):
C3 ────────╭─────────────  ← 基线漂移
           
电极接触不良 (硬件问题):
C3 ────╱╲╱╲╱╲────────────  ← 高频尖峰
```

## 📝 总结

### 核心概念速查

| 概念 | 关键点 |
|------|--------|
| **288 个试次** | 4 类任务 × 每类 72 次重复 = 288 个试次 |
| **事件标记** | 实验时实时写入 GDF 文件（769/770/771/772） |
| **分段** | 将连续数据切割成离散的 4 秒试次 |
| **Epochs** | (288, 22, 1000) ← 符合机器学习格式 |
| **特征提取** | 必须基于分段后的 epochs（不能直接用 raw） |
| **ICA vs 试次剔除** | 两级去噪：成分层面 + 试次层面 |

### 为什么这样做？

```
Q: 为什么有 288 个试次，不是只有 4 个动作吗？
A: 4 个动作类型 × 每个重复 72 次 = 288 个试次
   只有这样才有足够的样本训练分类器！

Q: 为什么必须分段？
A: 连续数据无法打标签，也不符合机器学习格式
   分段后：(288, 22, 1000) → 标准的 (N, F) 格式！

Q: 事件标记有什么用？
A: 自动提供 288 个试次的标签，无需手动标注！
   y = epochs.events[:, 2] ← 直接从 GDF 文件读取

Q: ICA 去噪后为什么还要剔除试次？
A: ICA 去除稳定伪迹（眨眼），试次剔除去除瞬时伪迹（咳嗽）
   两级防护，提高数据质量！
```
