# 模型训练与评估流程解析：`general_process.py` vs `test_on_evaluation_set.py`

在 BCI Competition IV 2a 数据集的处理流程中，我们编写了两个看似包含相似流程（预处理 -> 特征提取 -> 分类）但目标完全不同的核心脚本。理解它们的区别，对于正确执行单被试分析和独立跨期评估至关重要。

可以把它们形象地理解为：“**平时小测验**”与“**期末大考**”。

---

## 1. 核心区别总结一览

| 维度 | `general_process.py` (平时测验) | `test_on_evaluation_set.py` (期末大考) |
| :--- | :--- | :--- |
| **测试场景** | 闭环测试 / 模型内验证 | 跨域泛化测试 / 独立验证 |
| **所需数据** | 单一 Session (如 `A01T`) | 双 Session (`A01T` 训练 + `A01E` 评估) |
| **评估机制** | 10 折交叉验证 (K-Fold CV) | 独立测试预测 (Hold-out Test) |
| **标签来源** | MNE 解析 GDF 自带的 `epochs.events`  | 外部独立的官方 `true_labels/*.mat` 文件 |
| **Epoch 切分** | 根据类别事件标记 (`769-772`) 直接切分 | 根据 Trial 起点事件 (`768`) 切分并向后偏移 `+2s` |
| **工程意义** | 用于快速验证流程、对比特征优劣、调参 | 毕业论文中必须汇报的核心泛化性能指标 |

---

## 2. 深入解析：为什么需要两套独立的流程？

### 2.1 `general_process.py`（单 session 内部交叉验证）

- **目的**：验证整个预处理和分类处理管线是否连通，评估当前特征提取方案（CSP、小波、融合）是否有区分度。
- **机制**：
  - 它加载 `A01T.gdf`，数据自带了完整的事件标签（即 769=左手、770=右手、771=双脚、772=舌头）。
  - MNE 读取时，直接利用 769-772 标签进行 Epoch 试次切分（`tmin=0, tmax=4`，精准截取这四个动作发生的窗口）。
  - 然后将这 288 个提取出来的特征，利用 10折交叉验证（切分为10份，轮流用9份训练、1份测试），得到一个**平均准确率**。
- **结论**：它反馈的是模型对**同日收集数据**的拟合和同源数据的分类能力。

### 2.2 `test_on_evaluation_set.py`（跨 session 独立测试）

- **目的**：检验模型的**真实泛化能力**。脑电信号（EEG）具有强烈的非平稳性和时间变异性，同一个人在不同日期测出的脑电模式会有差异。这要求我们用跨日数据做“期末大考”。
- **机制的巨大难点（官方的“防作弊”设计）**：
  - 官方提供的评估集 `A01E.gdf` 内部**故意去掉了所有的 769-772 类别标签**！
  - 整个 `A01E.gdf` 里，标志着动作发生的地方，全都统一用了一个特殊的标记：`768`（代表 Trial 试次开始，但不知道要做什么动作）。
  - 真正的答案（标签），被单独发在了 `true_labels/A01E.mat` 文件里（里面全是 1, 2, 3, 4 的数字）。

---

## 3. 标签对齐与修正：`test_on_evaluation_set` 的特殊处理

为了解决上述“评估集无标签”的难题，我们在 `test_on_evaluation_set.py` 中做了特殊机制的重构：

### 1) 统一训练与测试的标签语言（底层归一）
如果训练集用原本 GDF 解析出的 `769-772` 进行训练，测试时却只有 `.mat` 的 `1, 2, 3, 4`，SVM 分类器将彻底失效，因为它遇到的是未见过的类别编号。
**解决方案**：
完全抛弃 MNE 解析处的中间产物（769-772），训练时也强制读取 `A01T.mat`（1,2,3,4），测试时读取 `A01E.mat`（1,2,3,4）。让 SVM 始终处于一个只认识 `1, 2, 3, 4` 的统一口径下。

### 2) 基于 `768` 试次起点进行时间偏移切分
因为 `A01E` 没有 `769-772` 的 cue（提示）标记，所以我们利用 `768` 这个试次唯一的共同起点事件来强制切分 Epochs。
**关键的时间逻辑还原：**
- 实验范式中，`768` 出现在 `t=0`。
- 运动提示箭头（原本的 `769-772`）出现在 `t=2s`。
- 我们需要获取的是提示箭头发生之后的 4 秒脑电波。
- **解决方案**：对 `768` 事件切分时，将起始与终点时间各自**延后 2 秒**。
  - 原本用 `769` 切分：截取 `[0, 4] 秒`。
  - 现在用 `768` 切分：截取 `[2, 6] 秒`。
  通过这种方法，我们在没有特定事件标签的盲测区域中，精准扣取出了目标运动想象的物理数据段。

### 3) 伪迹剔除后的动态对齐 (`epochs.selection`)
如果直接把解析出的 288 个脑电片段与 288 个 `.mat` 标签按顺序对拼，会由于 ICA 前后的“坏片段剔除”行为导致整个后续序列错位（如中间删掉了第3个片段，后续标签全部错位一格）。
**解决方案**：代码中使用 `epochs.selection` 获取到了残存脑电片段的精确源索引号，对 `.mat` 的标签数组进行了严格的数组下标映射提取（即 `aligned_labels = true_labels[epochs.selection]`），保障了被剔除试次后的每一条脑电数据都能与正确答案硬连接不脱靶。
