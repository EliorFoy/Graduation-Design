## 1. 数据集基础

### 1.1 BCIC IV-2a 概述

| 属性 | 值 |

| **实验类型** | 运动想象（Motor Imagery, MI） |
| **被试数量** | 9 人（A01-A09） |

## 2. 通道映射

### 2.1 为什么需要通道映射？

**问题根源**：BCIC IV-2a 的 GDF 文件头中通道名称信息缺失或不完整

```
原始通道名称：
['EEG-Fz', 'EEG', 'EEG', 'EEG', ..., 'EEG', 'EOG-left', 'EOG-central', 'EOG-right']
              ↑                                              ↑
         只有 1 个有名称                                  EOG 有名称
```

**后果**：MNE-Python 读取时自动生成重复的 `'EEG'` 名称，无法直接知道哪个通道是 C3、C4

### 2.2 社区标准通道顺序

这是 MOABB、Braindecode 及数千篇论文使用的标准顺序：

```python
EEG_CHANNELS = [
    # 额区 (Frontal)
    'Fz',
    
    # 额中央区 (Fronto-Central) - 从左到右
    'FC3', 'FC1', 'FCz', 'FC2', 'FC4',
    
    # 中央区 (Central) - 从左到右
    'C5', 'C3', 'C1', 'Cz', 'C2', 'C4', 'C6',
    
    # 中央顶区 (Centro-Parietal) - 从左到右
    'CP3', 'CP1', 'CPz', 'CP2', 'CP4',
    
    # 顶区 (Parietal) - 从左到右
    'P1', 'Pz', 'P2',
    
    # 顶枕区 (Parieto-Occipital)
    'POz'
]
# 共 22 个 EEG 通道

EOG_CHANNELS = ['EOG-left', 'EOG-central', 'EOG-right']
```

### 2.3 空间布局（10-20 系统）

```
                    Fz
                FC3 FC1 FCz FC2 FC4
              C5 C3 C1 Cz C2 C4 C6
                  CP3 CP1 CPz CP2 CP4
                      P1 Pz P2 POz
```

**记忆技巧**：
- **从前到后**：F → FC → C → CP → P → PO
- **从左到右**：数字 3/5 = 左侧，z = 中线，2/4 = 右侧
- **关键通道**：
  - `C3`：左手运动区（右手想象时激活）
  - `C4`：右手运动区（左手想象时激活）
  - `Cz`：双脚运动区

### 2.4 验证映射正确性

**方法 1：α波枕叶优势验证**
```python
# 原理：闭眼静息时，枕叶视觉皮层的α波 (8-13Hz) 功率应显著高于前额
# 预期：枕叶/前额α功率比 > 1.5

occipital = ['POz', 'Pz', 'P1', 'P2']
frontal = ['Fz', 'FCz']
ratio = occ_power / front_power
print(f"枕叶/前额α功率比：{ratio:.2f}")  # 预期：> 1.5
```

**方法 2：运动想象 ERD 验证**
```python
# 原理：
# - 想象右手 → 左侧 C3 通道μ/β功率下降
# - 想象左手 → 右侧 C4 通道μ/β功率下降
```

## 3. 预处理流程

### 3.1 完整流程图

```
原始 GDF 数据 (25 通道连续记录)
    ↓
Step 1: 加载 GDF 文件
    ↓
Step 2: 通道映射（应用社区标准顺序）
    ↓
Step 3-4: 轻度滤波 (1-40Hz, FIR + zero-phase)
    ↓
Step 5-6: ICA 分解 + 去除 EOG 成分
    ↓
Step 7: 任务滤波 (8-30Hz，聚焦μ+β节律)
    ↓
Step 8: 重参考（乳突参考 → 平均参考）
    ↓
Step 9: 分段（根据事件标记切割为 0-4 秒试次）
    ↓
Step 10: 剔除伪迹试次（移除 1023 标记的试次）
    ↓
最终 epochs_final (280, 22, 1000) ← 干净的、可用于分类的数据
```

### 3.2 FIR 滤波器详解

**为什么使用 FIR 而不是 IIR？**

| 特性 | FIR | IIR |
|------|-----|-----|
| 相位响应 | **严格线性相位** | 非线性相位 |
| 稳定性 | **绝对稳定** | 可能不稳定 |
| 计算量 | 较大 | 较小 |
| 群延迟 | 恒定 | 频率相关 |

**MNE 代码**：
```python
raw.filter(
    l_freq=1.0,      # 高通截止 1Hz：去除慢漂移
    h_freq=40.0,     # 低通截止 40Hz：保留μ+β节律
    method='fir',    # 使用 FIR 保证线性相位
    phase='zero',    # 零相位滤波，避免时序偏移
    verbose=False
)
```

**零相位滤波原理**（filtfilt）：
- 前向 - 反向滤波 → 相位响应抵消
- 幅度响应平方：$|H|^2$，截止更陡峭
- 相位响应：$\phi(\omega) + (-\phi(\omega)) = 0$ → **零相位失真**

### 3.3 重参考（Re-reference）

**为什么要做重参考？**

EEG 信号测量的是**相对电压差**，需要一个参考点。如果参考电极本身有噪声，会污染所有通道！

**平均参考原理**：
```python
# 计算所有 EEG 通道的平均值作为新参考
V_new = V_channel - mean(V_all_channels)

# 示例（简化到 3 个通道）:
原始值: C3: 10 μV, Cz: 15 μV, C4: 8 μV
平均值 = (10 + 15 + 8) / 3 = 11 μV

重参考后:
  C3: 10 - 11 = -1 μV
  Cz: 15 - 11 = 4 μV
  C4: 8 - 11 = -3 μV
```

**优势**：
- ✅ 无偏参考：不依赖单个电极，减少参考电极噪声
- ✅ 标准做法：运动想象研究的常用方法
- ✅ 提高信噪比：突出局部脑活动

### 3.4 分段（Epoching）

**为什么要分段？**

从连续数据到离散试次，便于机器学习处理：

```
连续 EEG 数据（Continuous Raw Data）:
┌─────────────────────────────────────────┐
│ ──────┬───┬─────┬───┬─────┬───┬────── │
│       ↑   ↑     ↑   ↑     ↑   ↑        │
│      T1  T2    T3  T4    T5  T6        │
└─────────────────────────────────────────┘
          ↓ 根据事件标记切割 ↓
分段后的 Epochs:
┌──────────┐  ┌──────────┐  ┌──────────┐
│  Trial 1 │  │  Trial 2 │  │  Trial 3 │
│  0-4 秒  │  │  0-4 秒  │  │  0-4 秒  │
│  (左手)  │  │  (右手)  │  │  (双脚)  │
└──────────┘  └──────────┘  └──────────┘
```

**数据结构变化**：
```python
# 分段前：Continuous Raw
raw.shape = (22, 360000)  # 24 分钟连续数据

# 分段后：Epochs
epochs.shape = (288, 22, 1000) 
               ↑    ↑   ↑
             试次数 通道 4 秒×250Hz
```

**为什么选择 4 秒？**
- ✅ 包含视觉处理 (~0.3s)
- ✅ 包含反应准备 (~0.5s)
- ✅ 包含持续想象 (2-3s)
- ✅ 避免包含过多噪声和受试者走神

### 3.5 伪迹试次剔除

**为什么 ICA 去噪后还要剔除试次？**

| 步骤 | 剔除对象 | 伪迹类型 | 操作层面 |
|------|---------|---------|---------|
| **ICA 去噪** | 独立成分 | **稳定的、周期性的**伪迹 | **成分层面** |
| **剔除试次** | 整个 Epoch | **瞬时的、偶发的**严重伪迹 | **试次层面** |

**ICA 能去除的**：眨眼、眼动、心跳（稳定出现）

**ICA 不能去除的**：突然咳嗽、吞咽、电极接触不良（瞬时剧烈）

**代码逻辑**：
```python
# 找到所有标记为 1023 的事件
artifact_mask = events[:, 2] == 1023

# 剔除这些伪迹试次
epochs.drop_events(artifact_mask)

# 结果：原本 288 个试次 → 剩余 280 个
```

## 4. ICA 去噪详解

### 4.1 EOG 通道的作用机制

**核心结论**：ICA 可以在没有 EOG 通道的情况下去除伪迹，但**有 EOG 通道效果更佳、流程更自动化**。

**EOG 的双重作用**：

#### 作用 1：参与数据矩阵构建（ICA 分解时）
```python
# raw_ica 包含所有通道（22 EEG + 3 EOG = 25 通道）
ica.fit(raw_ica)

# 输入数据矩阵 X (25×T)：
X = [EEG_1(t), ..., EEG_22(t), EOG-left(t), EOG-central(t), EOG-right(t)]^T

# PCA 降维到 22 个主成分
# 眼电信号的强方差信息被保留在主成分中
```

#### 作用 2：作为参考标签（伪迹识别时）
```python
# find_bads_eog() 的核心逻辑:
eog_indices, eog_scores = ica.find_bads_eog(raw_ica, threshold=2.0)

# 内部计算：
for ic_index in range(n_components):
    ic_signal = ica.get_source(raw_ica)[ic_index]
    corr = pearsonr(ic_signal, eog_signal)
    
    if abs(corr) > threshold_value:
        eog_indices.append(ic_index)  # 标记为眼电成分
```

### 4.2 threshold 参数详解

**threshold 的作用时机**：在 ICA 分解**完成后**使用

**动态阈值计算**：
```python
# 1. 计算每个 ICA 成分与 EOG 通道的相关系数
correlations = [0.85, 0.12, 0.03, 0.67, ..., 0.45]

# 2. 计算阈值（动态阈值！）
mean_corr = np.mean(correlations)   # 平均值
std_corr = np.std(correlations)     # 标准差
threshold_value = mean_corr + threshold * std_corr
#                        ↑
#                   这里的 2.0 就是参数 eog_threshold

# 示例：
# mean_corr = 0.15, std_corr = 0.18
# threshold_value = 0.15 + 2.0 × 0.18 = 0.51

# 3. 标记超过阈值的成分
eog_indices = [idx for idx, corr in enumerate(correlations) if corr > threshold_value]
# 结果：[0, 3, ...]
```

**调参建议**：
| threshold | 检测灵敏度 | 假阳性风险 | 适用场景 |
|-----------|-----------|-----------|----------|
| **1.0** | 非常高 | 高 | 伪迹很微弱 |
| **1.5** | 较高 | 中等 | 一般情况 |
| **2.0** (默认) | 适中 | 低 | **推荐** |
| **2.5** | 较低 | 很低 | 伪迹非常明显 |
| **3.0** | 非常低 | 极低 | 只检测极端伪迹 |

### 4.3 官方文档要求

根据 BCIC IV-2a 官方文档 `desc_2a.pdf`：

> "The EOG channels are provided for the **subsequent application of artifact processing methods** and must not be used for classification."

> "Since three EOG channels are provided, **it is required to remove EOG artifacts** before the subsequent data processing."

**解读**：
- ✅ EOG 专门用于伪迹处理
- ❌ 分类器输入时不能包含 EOG（去噪后要剔除）
- ✅ 官方明确要求去除眼电伪迹

## 5. 小波变换

### 5.1 核心概念澄清

🔑 **db4 ≠ 分四层** —— 这是两个**独立**的概念！

| 概念 | 含义 | 符号 |
|------|------|------|
| **db4** | Daubechies 小波族，阶数 N=4（4 个消失矩） | 小波类型 |
| **分解层数** | 小波分解的层级数（level） | 分解参数 |

**你可以用 db4 小波做 1 层、3 层、5 层...任意层分解！**

### 5.2 db4 小波的数学定义

**消失矩（Vanishing Moments）**：4 个
$$\int_{-\infty}^{\infty} t^k \psi(t) dt = 0, \quad k = 0, 1, 2, 3$$

这意味着 db4 可以**压缩掉 3 次及以下**的多项式信号。

**滤波器长度**：$2N = 8$ 个系数
- 低通分解滤波器 $h[n]$：8 个系数
- 高通分解滤波器 $g[n]$：8 个系数

**db4 的特点**：
- 比 db1 (Haar) 更平滑，适合分析连续信号
- 比高阶 db（如 db10）计算更快，时域支撑更短
- **适合分析 EEG 信号**（如 BCIC IV-2a 数据）

### 5.3 小波分解原理（Mallat 算法）

```
原始信号：x[n] (长度 L)
    ↓ [低通滤波 + 下采样 2]  [高通滤波 + 下采样 2]
A1 (近似) (长度 L/2)       D1 (细节) (长度 L/2)
    ↓
A2 (长度 L/4)              D2 (长度 L/4)
    ↓
A3 (长度 L/8)              D3 (长度 L/8)
    ↓
A4 (长度 L/16)             D4 (长度 L/16)
```

**数学表达**：
- 第 $j$ 层近似系数：$A_j[k] = \sum_n x[n] \cdot \phi_{j,k}(n)$
- 第 $j$ 层细节系数：$D_j[k] = \sum_n x[n] \cdot \psi_{j,k}(n)$

其中 $\phi_{j,k}$ 是尺度函数，$\psi_{j,k}$ 是小波函数。

### 5.4 为什么 EEG 分析常选 4 层分解？

**频段对应关系**（假设采样率 $f_s = 250$ Hz）：

| 分解层 | 频段范围 | 对应节律 | 生理意义 |
|--------|---------|---------|---------|
| **D1** | 62.5-125 Hz | γ (高频) | 肌电噪声为主 |
| **D2** | 31.25-62.5 Hz | γ (低频) | 高频脑活动 |
| **D3** | 15.6-31.25 Hz | **β** | **运动想象** |
| **D4** | 7.8-15.6 Hz | **μ + β** | **运动想象核心** |
| **A4** | 0-7.8 Hz | δ + θ | 慢波、漂移 |

**选择 4 层的理由**：
1. ✅ **覆盖关键频段**：D3+D4 恰好覆盖 7.8-31.25 Hz（μ+β节律）
2. ✅ **避免过度分解**：D1、D2 主要是高频噪声
3. ✅ **计算效率**：4 层已足够，更多层信息增益有限

**数学验证**：
对于 $f_s = 250$ Hz，第 $j$ 层细节系数的频段为：
$$f \in \left[\frac{f_s}{2^{j+1}}, \frac{f_s}{2^j}\right]$$

- **D4 层**（$j=4$）：$f \in [7.8125, 15.625]$ Hz → 完美覆盖 μ 节律！
- **D3 层**（$j=3$）：$f \in [15.625, 31.25]$ Hz → 覆盖 β 节律！

### 5.5 代码示例

```python
import pywt
import numpy as np

signal = np.random.randn(1000)  # 1000 采样点
fs = 250  # 采样率 250 Hz

# 使用 db4 小波进行 4 层分解
wavelet = 'db4'
level = 4
coeffs = pywt.wavedec(signal, wavelet, level=level)

# coeffs 结构：[A4, D4, D3, D2, D1]
A4, D4, D3, D2, D1 = coeffs

print(f"A4 (0-7.8 Hz): 长度 {len(A4)}")
print(f"D4 (7.8-15.6 Hz, μ): 长度 {len(D4)}")
print(f"D3 (15.6-31.25 Hz, β): 长度 {len(D3)}")

# 提取特征：各层能量
features = [np.sum(np.abs(c)**2) for c in coeffs]
```

### 5.6 CWT vs DWT

| 特性 | 连续小波变换 (CWT) | 离散小波变换 (DWT) |
|------|-------------------|-------------------|
| **函数** | `pywt.cwt()` | `pywt.wavedec()` |
| **尺度参数** | 连续变化 | 离散变化 ($a = 2^j$) |
| **输出形式** | **时频图** (矩阵) | **系数向量** |
| **冗余度** | 高冗余 | 低冗余（临界采样） |
| **计算量** | 大 | 小（快速 Mallat 算法） |
| **典型应用** | 时频可视化 | **特征提取**、分类 |

**为什么 EEG 分类用 DWT？**
1. ✅ **特征维度压缩**：4 层分解 → 仅需 5 个能量特征
2. ✅ **频带与生理节律天然对齐**：D4 层就是 μ 节律
3. ✅ **去噪便利**：可直接置零某一层系数后重构

**建议**：
- 做**可视化分析**（写论文画图）：用 **CWT**
- 做**分类模型**：用 **DWT (分层)**

### 5.7 为什么一层分解不够？

**一层分解的实质**：
- A1: $[0, 62.5]$ Hz ← μ节律 (8-13Hz) 被埋在这里面，与δ、θ混在一起
- D1: $[62.5, 125]$ Hz ← 空（没有目标信号）

**四层分解**：
- A4: $[0, 7.8]$ Hz
- D4: $[7.8, 15.6]$ Hz ← **单独捕获 μ 节律**
- D3: $[15.6, 31.25]$ Hz ← **单独捕获 β 节律**
- D2, D1: 高频噪声

**结论**：一层分解**能覆盖全频段，但不能分辨细分节律**，无法满足 EEG 分类需求。

## 6. 特征提取与分类

### 6.1 数据结构变化总览

```python
# 1. 原始连续数据
raw: (25, 360000) ← 24 分钟连续记录

# 2. 预处理后分段
epochs: (288, 22, 1000)
  ├─ 288 个试次（样本）
```

**❌ 不分段的问题**：

| 概念 | 关键点 |
|------|--------|
| **288 个试次** | 4 类任务 × 每类 72 次重复 = 288 个试次 |

## ❓ 常见问题解答

**Q1: 为什么有 288 个试次，不是只有 4 个动作吗？**  
A: 4 个动作类型 × 每个重复 72 次 = 288 个试次。只有这样才有足够的样本训练分类器！

**Q2: 为什么必须分段？**  
A: 连续数据无法打标签，也不符合机器学习格式。分段后：(288, 22, 1000) → 标准的 (N, F) 格式！

**Q3: 事件标记有什么用？**  
A: 自动提供 288 个试次的标签，无需手动标注！`y = epochs.events[:, 2]` ← 直接从 GDF 文件读取。

**Q4: ICA 去噪后为什么还要剔除试次？**  
A: ICA 去除稳定伪迹（眨眼），试次剔除去除瞬时伪迹（咳嗽）。两级防护，提高数据质量！

**Q5: db4 是分四层吗？**  
A: ❌ 不是。db4 是小波类型（4 消失矩），层数是独立参数。你可以用 db4 做任意层分解。

**Q6: 为什么一层分解不够？**  
A: 一层分解能覆盖全频段，但不能分辨细分节律。μ(8-13Hz) 和 β(13-30Hz) 会混在同一个低频系数里，导致分类器无法学习。
