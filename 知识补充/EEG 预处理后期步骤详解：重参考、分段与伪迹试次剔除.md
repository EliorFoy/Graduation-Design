# EEG 预处理后期步骤详解：重参考、分段与伪迹试次剔除

## 概述

EEG 预处理的后期处理阶段（Step 7-9）对最终分类结果至关重要，包括：
1. **重参考**（Re-reference）：改善参考电极质量
2. **分段**（Epoching）：从连续数据到离散试次
3. **伪迹试次剔除**：提高数据质量

---

## 一、Step 7：重参考（Re-reference）

### 为什么要做重参考？

EEG 信号测量的是**相对电压差**，需要一个参考点：

```python
# 原始数据：以乳突（Mastoid）为参考
# 每个 EEG 通道记录的是相对于参考电极的电压
V_channel = V_scalp - V_reference
```

**问题：** 如果参考电极本身有噪声，会污染所有通道！

### 平均参考（Average Reference）

```python
raw_final = set_reference(raw_final, ref='average')
```

**原理：**
```python
# 计算所有 EEG 通道的平均值作为新参考
V_new = V_channel - mean(V_all_channels)

# 示例（简化到 3 个通道）:
原始值:
  C3: 10 μV
  Cz: 15 μV  
  C4: 8 μV
  
平均值 = (10 + 15 + 8) / 3 = 11 μV

重参考后:
  C3: 10 - 11 = -1 μV
  Cz: 15 - 11 = 4 μV
  C4: 8 - 11 = -3 μV
```

**优势：**
- ✅ **无偏参考**：不依赖单个电极，减少参考电极噪声
- ✅ **标准做法**：运动想象研究的常用方法
- ✅ **提高信噪比**：突出局部脑活动

### 数学表达

```python
# 原始信号
V_i(t) = V_brain_i(t) + V_noise_ref(t)
              ↑             ↑
           真实脑电      参考电极噪声

# 重参考后
V_new_i(t) = V_i(t) - mean(V_all(t))
            ≈ V_brain_i(t) - mean(V_brain_all(t))
            (参考噪声被抵消了！)
```

### 类比理解

```
想象你在测量山的高度:

原始参考（乳突参考）:
  • 以海平面为 0 点
  • 但海平面有潮汐波动 → 引入误差
  
重参考（平均参考）:
  • 以所有山峰的平均高度为 0 点
  • 消除了"海平面波动"的影响
  • 更能反映每座山的相对高度
```

## 二、Step 8：分段（Epoching）

### 为什么要分段？

**从连续数据到试次（Trial）：**

```
连续 EEG 数据（Continuous Raw Data）:
┌─────────────────────────────────────────┐
│ ──────┬───┬─────┬───┬─────┬───┬────── │
│       ↑   ↑     ↑   ↑     ↑   ↑        │
│      T1  T2    T3  T4    T5  T6        │
│       ← 一个试次 (Trial/Epoch) →       │
└─────────────────────────────────────────┘

↓ 根据事件标记 (Event Markers) 切割 ↓

分段后的 Epochs:
┌──────────┐  ┌──────────┐  ┌──────────┐
│  Trial 1 │  │  Trial 2 │  │  Trial 3 │
│  0-4 秒  │  │  0-4 秒  │  │  0-4 秒  │
│  (左手)  │  │  (右手)  │  │  (双脚)  │
└──────────┘  └──────────┘  └──────────┘
```

### 参数含义

```python
create_epochs(raw_final, tmin=0, tmax=4)
                      ↑       ↑
                  刺激前   刺激后
                  0 秒    4 秒
                  
# 总时长 = tmax - tmin = 4 秒
# 采样率 250Hz → 每个试次 1000 个时间点
```

### BCIC IV-2a 数据集的实验范式

```
时间轴:
  0s     2s          4s         6s
  │------│-----------│----------│
  │      │           │          │
  固定   任务提示     执行想象    休息
  十字   (箭头)      (运动想象)
  
  ↓ 我们截取这段 ↓
      [tmin=0 到 tmax=4]
      包含完整的任务执行期
```

### 数据结构变化

```python
# 分段前：Continuous Raw
raw.shape = (n_channels, n_samples)
# 例如：(22, 360000) ← 24 分钟连续数据

# 分段后：Epochs
epochs.shape = (n_epochs, n_channels, n_times)
# 例如：(288, 22, 1000) 
#        ↑    ↑   ↑
#     试次数 通道 4 秒×250Hz
```

### 每个 Epoch 的结构（0-4 秒）

```
┌───────────────────────────────┐
│ 0s    1s    2s    3s    4s   │
│ │     │     │     │     │    │
│ ├─────┤─────┤─────┤─────┤    │
│       ↑                     │
│    任务提示                  │
│    (箭头出现)                │
│                             │
│       ← 运动想象执行期 →     │
│                             │
└───────────────────────────────┘

事件标记:
  769: 左手想象
  770: 右手想象  
  771: 双脚想象
  772: 舌头想象
```

### 为什么需要分段？

#### 1. 实验设计需求

```
实验流程时间线:
  0s     2s          4s         6s        8s
  │------│-----------│----------│---------│
  │      │           │          │         │
  固定   任务提示     执行想象   休息     下一个
  十字   (箭头→)     (右手)              试次
  
  ↓ 我们只关心这段 ↓
      [0-4 秒] ← 一个 Epoch/Trial
```

**分段的作用：**
- ✅ **提取感兴趣的时间段**：只保留任务执行期
- ✅ **排除无关数据**：休息期、准备期不要
- ✅ **对齐刺激**：所有试次都从刺激出现开始计时

#### 2. 机器学习需求

分类器需要的数据格式：

```python
# ❌ 不分段：无法训练
X = raw_data  # (22, 360000) ← 怎么打标签？
y = ???       # 每个时间点是什么标签？

# ✅ 分段后：可以训练
X = epochs.get_data()  # (288, 22, 1000)
#                        ↑    ↑   ↑
#                      样本数 通道 时间点

y = labels  # (288,) ← 每个试次一个标签
            # [769, 770, 769, 771, 770, 772, ...]
            #  左手，右手，左手，双脚，右手，舌头...
```

**类比理解：**
```
想象你在训练一个图像分类器:

❌ 不分段:
   给模型看一整部电影 → "这是喜剧还是悲剧？"
   （太长了，信息混杂，无法判断）

✅ 分段:
   给模型看一张张截图 → "这张图是笑还是哭？"
   （每张图一个标签，清晰明确）
```

### 分段的工作原理

```python
# MNE 内部逻辑（简化版）
def create_epochs(raw, tmin, tmax):
    """
    根据事件标记切割连续数据
    """
    # 1. 找到所有事件标记的位置
    events = mne.events_from_annotations(raw)
    # events.shape = (n_events, 3)
    # 每行：[时间点数，0, 事件代码]
    
    # 示例:
    events = [
        [1000, 0, 769],   # 第 1000 个采样点，左手
        [3500, 0, 770],   # 第 3500 个采样点，右手
        [6000, 0, 771],   # 第 6000 个采样点，双脚
        ...
    ]
    
    # 2. 对每个事件，截取 [tmin, tmax] 的数据
    epochs_list = []
    for event_time, _, label in events:
        start_idx = event_time + int(tmin * sfreq)
        end_idx = event_time + int(tmax * sfreq)
        
        epoch_data = raw[:, start_idx:end_idx]
        epochs_list.append(epoch_data)
    
    # 3. 堆叠成 Epochs 对象
    epochs = EpochsArray(epochs_list)
    # shape: (n_events, n_channels, n_times)
    
    return epochs
```

### 为什么选择 4 秒？

```
运动想象的神经响应时间:
  0s     0.5s   1s    2s    3s    4s
  │------│------│-----│-----│-----│
  │      │      │     │     │     │
  刺激   视觉   决策  持续  持续  结束
  出现   处理   开始  想象  想象
  
  ├────── 需要覆盖的完整过程 ──────┤
  
  4 秒足够长:
  ✓ 包含视觉处理 (~0.3s)
  ✓ 包含反应准备 (~0.5s)
  ✓ 包含持续想象 (2-3s)
  
  4 秒不太长:
  ✓ 避免包含过多噪声
  ✓ 避免受试者走神
  ✓ 计算量适中
```

### 其他常见分段参数选择

```python
# 方案 A: 包含准备期（基线校正）
create_epochs(raw, tmin=-1, tmax=4)
#              ↑
#         刺激前 1 秒（基线期）

# 方案 B: 只关注特定阶段
create_epochs(raw, tmin=2, tmax=4)
#                           ↑
#                      只取执行期的后半段

# 方案 C: 更长观察窗
create_epochs(raw, tmin=0, tmax=6)
#                           ↑
#                      包含部分休息期（研究恢复过程）
```

## 三、Step 9：剔除伪迹试次

### 为什么 ICA 去噪后还要剔除试次？

```python
events, _ = mne.events_from_annotations(raw_final)
epochs_final = drop_artifact_epochs(epochs, events)
```

**原因：**
- ICA 去除了**大的、稳定的**伪迹（眨眼、眼动、心跳）
- 但可能还有**瞬时的、残留的**伪迹：
  - 突然的肌肉收缩
  - 吞咽动作
  - 电极接触不良导致的尖峰

### 正常试次 vs 含伪迹试次

```
正常试次:
C3 ────────────────────────────
Cz ────────────────────────────
C4 ────────────────────────────

含伪迹的试次:
C3 ──────────╱╲──────────────  ← 肌肉尖峰
Cz ─────╭────╯╰────╮─────────  ← 眼动漂移
C4 ────────────────────────────

→ 这种试次会影响分类器训练！
```

### BCIC IV-2a 的特殊标记

```python
artifact_codes=[1023]  # 官方伪迹标记代码
```

**官方数据集的事件代码：**
```python
# 运动想象任务:
  769: 左手
  770: 右手
  771: 双脚
  772: 舌头

# 伪迹标记:
  1023: 严重伪迹试次（由专家手动标记）
```

### 检测逻辑

```python
# 找到所有标记为 1023 的事件
artifact_mask = events[:, 2] == 1023
#                 ↑         ↑
#              事件数组   事件代码列

# 示例:
events = [
    [1000, 0, 769],    # 1000ms 处，左手试次
    [2500, 0, 770],    # 2500ms 处，右手试次
    [4000, 0, 1023],   # 4000ms 处，伪迹！← 要剔除
    [5500, 0, 771],    # 5500ms 处，双脚试次
]

artifact_mask = [False, False, True, False]

# 剔除这个伪迹试次
epochs.drop_events(artifact_mask)

结果：原本 288 个试次 → 剩余 287 个
```


```latex
用于后续的特征提取和分类器训练。
```

## 八、常见问题解答

### Q1: 为什么 ICA 去噪后还要剔除试次？

**A:** ICA 去除的是**稳定的、可建模的**伪迹源。但瞬时伪迹（如突然的肌肉收缩）需要直接剔除整个试次。

### Q2: 1023 是谁标记的？

**A:** 数据集采集时的**实验专家**人工标记的。他们在被试实验过程中发现明显伪迹就按下标记按钮。

### Q3: 能不能不剔除，直接用所有试次？

**A:** 可以，但会**降低分类准确率**。建议对比一下：

```python
# 方案 A: 剔除伪迹试次
epochs_clean = drop_artifact_epochs(epochs, events)
accuracy_A = train_and_test(epochs_clean)  # 通常更高

# 方案 B: 保留所有试次
accuracy_B = train_and_test(epochs)  # 可能较低
```

### Q4: 还有其他剔除方法吗？

**A:** 有的！还可以基于：
- 幅度阈值（如 > 100μV 的试次）
- 方差阈值（异常波动的试次）
- 自动检测算法（如自相关、谱分析）

## 九、分段后的应用

### 特征提取

```python
# CSP 特征提取（你的毕设要用到）
for epoch in epochs:  # 逐个试次处理
    X = epoch.get_data()  # (22, 1000)
    
    # 应用空间滤波
    X_filtered = W @ X  # (22, 1000)
    
    # 计算方差作为特征
    feature = np.var(X_filtered, axis=1)
    
    # 一个试次 → 一个特征向量 (22,)
```

### 分类器训练

```python
# 准备训练数据
X = epochs.get_data()  # (288, 22, 1000)
y = labels             # (288,) ← 每个试次的标签

# 展平数据（对于某些分类器）
X_flat = X.reshape(288, -1)  # (288, 22×1000)

# 训练 SVM
clf.fit(X_flat, y)
```

## 十、总结

这三个步骤是将**连续的、含噪的**EEG 数据转换为**离散的、干净的、可用于机器学习**的分类数据的关键环节：

1. **重参考**：改善参考质量，提高信噪比
2. **分段**：组织成试次格式，便于机器学习处理
3. **伪迹试次剔除**：进一步提高数据质量

**类比总结：**
```
Raw = 24 分钟电影（信息混杂）
Epochs = 288 个精彩片段（每个 4 秒）
Epochs_final = 280 个高质量片段（剔除瑕疵）
```
